Signup Flow

  • Unauthenticated users see the login screen on every page load
  • Login form: email + password, submits via NextAuth signIn("credentials")
  • "Create one" link toggles to the registration form (no page navigation)
  • Registration: display name, email, password, optional Venmo handle
  • On success, redirect to login with a success banner (no auto-login)
  • On error, inline error message (duplicate email, wrong invite code, weak password)

Session Behavior

  • Sessions use NextAuth JWT strategy — persists across refresh, tabs, browser restart
  • Session max age: 7 days
  • Logged-in user's display name shown as a badge in the header
  • "Log Out" button calls signOut() and returns to login screen

Validation Rules

  • Email: must be unique, valid format
  • Password: minimum 8 characters
  • Display name: required, non-empty
  • All validation errors shown inline above the form

Server-side Changes

  • New users table: id, name, email, password_hash, venmo_handle, is_commissioner, created_at
  • New POST /api/register endpoint (hashes password with bcryptjs, stores optional Venmo handle)
  • New pages/api/auth/[...nextauth].js (credentials provider + JWT)
  • All API mutations switch from x-league-secret header to getServerSession()
  • Delete old pages/api/auth.js, remove LEAGUE_SECRET / COMMISSIONER_NAME env vars

Identity Changes in Client

  • Remove getSecret(), getMyName(), saveMyName(), clearMyName()
  • Remove sessionStorage / localStorage identity code
  • Replace with useSession() from next-auth/react
  • Draft join uses authenticated user's name automatically (no self-reported input)
  • Wrap _app.js with <SessionProvider>

Security

  • Passwords hashed with bcryptjs cost factor 12
  • NEXTAUTH_SECRET generated with openssl rand -base64 32
  • All DB writes use parameterized sql tagged template (no string concatenation)
  • Registration validates CSRF token from NextAuth cookie
  • No email verification (trusted friends league ~12 users)
  • No password reset (commissioner can reset via Neon console)

New Env Vars

  • NEXTAUTH_SECRET — JWT signing key
  • NEXTAUTH_URL — canonical app URL

Remove: LEAGUE_SECRET, COMMISSIONER_NAME

Migration

  • First deploy: users table empty, commissioner registers first
  • Commissioner manually sets is_commissioner = true via Neon console (or seed script)
  • Existing league_state JSONB unaffected — app resets each tournament
  • Drafter list stays as name strings in state, but also stores userId for future relational queries
